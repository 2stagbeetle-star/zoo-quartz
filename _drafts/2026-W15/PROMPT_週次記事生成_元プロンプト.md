# 週次記事生成スキル（強化版）

## 概要
毎週1回、動物園・水族館・保全・獣医学・生態学など自然科学全般の関連ニュースを網羅的に収集し、深掘り調査を行ったうえで**「週次総合まとめ」「個別記事群（A/B/Cランク別）」「短縮版」の3層構造**で記事を出力する。
**時間は制限せず、可能な限り一次資料まで遡って裏取りを行い、推測を排した正確で質の高い記事を量産すること**を最優先とする。

---

## ステップ 1: 実行準備と保存用フォルダ作成

記事をまとめて出力するため、その週専用のフォルダを作成する。

```bash
WEEK=$(date +%V)
YEAR=$(date +%Y)
TARGET_DIR="_drafts/${YEAR}-W${WEEK}"
mkdir -p "$TARGET_DIR"
```

---

## ステップ 2: ニュース収集と調査の強化方針

以下の手順で過去1週間の関連ニュースを徹底的に収集・調査する。

**【トピック網羅性の確保】**
特定のカテゴリに偏らないよう、以下の全セグメントから最低1件以上のトピックを抽出する。
1. **[運営・社会]**: 園館経営、法改正、地域連携、求人動向、施設改修、JAZA/AZA動向。
2. **[技術・現場]**: 飼育技術（給餌・環境）、獣医療（手術・疾病）、エンリッチメント、繁殖管理（人工哺育等）。
3. **[学術・研究]**: 最新論文（Google Scholar/arXiv）、新種発見、系統地理、ゲノム解析、分類変更。
4. **[環境・保全]**: 地球規模の気候変動影響、海洋プラスチック汚染、生息地再生、野生復帰プロジェクト、CITES動向。
5. **[種別バラエティ]**: 哺乳類だけでなく、鳥類・爬虫類・両生類・魚類・無脊椎動物・植物を意識的に含める。

**【一次資料到達の徹底】**
- **一次資料**: 公式サイトお知らせ、プレスリリース、園館/研究機関/協会/行政の公式発表、論文・学会発表、公式SNS原投稿。
- **準一次資料**: 公式関係者コメントを含む報道、自治体・指定管理者資料、公式ブログ/YouTube。

**【深掘り調査ルール・参照範囲・記事作成時の必須条件】**
`CLAUDE.md` の「リサーチ運用ルール（共通方針）」セクションに従うこと。

---

## ステップ 3: 記事分類とランク判定（生成戦略）

収集したニュースを以下の戦略に基づいて分類・ランク付けする。
単一のニュースから必要に応じて**「総合記事」「速報短報」「現場目線記事」「SNS転用版」**など複数フォーマットを派生作成し、本数を最大化する。

### 記事ランク制
- **Aランク記事**: 一次資料・裏取り複数あり。背景調査済み。**飼育員目線の示唆が豊富。**（長文で正式記事化。インフォグラフィック生成対象）
- **Bランク記事**: 一次資料または信頼できる報道あり。主要事実・背景確認済み。（通常記事として掲載）
- **Cランク記事**: 速報性重視。事実は一定確認済みだが詳細は限定的。（短報として掲載。後日更新可能）
※事実が不明瞭なものは一切掲載しない。

### カテゴリ分類（本数最大化のための戦略）
以下の視点から小さな話題も記事として拾う。
重大ニュース / 搬入搬出 / 繁殖・人工哺育 / 展示変更・公開 / 海外技術・福祉 / 保全教育・地域連携 / 高齢個体ケア / 研究・論文発表 / 事故・感染症 / 運営・施設改修

---

## ステップ 4: 記事の生成出力（3層構造）

収集・判定したニュースをもとに、以下の3層のマークダウンファイルを `$TARGET_DIR/` に生成する。

### 【第1層】週次総合まとめ
該当週の全体像と傾向を整理した1本を見出し記事として作成する。
- **ファイル名**: `$TARGET_DIR/週次まとめ_${YEAR}-W${WEEK}.md`
- **内容**: 今週の主要動向 / 分類別一覧（A/B/Cランク明記） / 全体傾向 / **現場への示唆**

### 【第2層】個別記事群（1ニュース1記事以上）
Aランク・Bランク・Cランク別に個別のニュース記事（.md）を作成する。
重要（A）は長め、軽微（C）は短報とする。
- **ファイル名フォーマット**: `$TARGET_DIR/[ランク]_[カテゴリ略称]_[YYYYMMDD]_[キーワード].md`
- **既存記事確認**: `find ./0* -name "*.md" | xargs grep -h "^title:"` にて既存記事との重複・既報を確認し、必要なら過去動向を整理・言及する。

**📝 A・Bランク向け 推奨フォーマット（詳細版）**
```markdown
---
title: "[記事タイトル]"
date: YYYY-MM-DD
updated: YYYY-MM-DD
tags:
  - [カテゴリ・タグ]
author: "Zoo Knowledge Vault"
draft: true
---

## 基本情報
- 施設名: / 国・地域:
- 動物種: (分類:)
- 公表日: / 重要度: [A/B] / 信頼度: [一次資料確認済 等]

## 概要
（何が起きたかを簡潔に整理）

## 詳細
- 事実関係・経緯・背景
- 関連発表・他媒体との差分（あれば）
- 続報の有無

## 飼育員・飼育スタッフ目線の注目点
（飼育管理、展示運営、福利、来園者対応など、現場での示唆を記述）

## 関連する論点
（種保存、福祉、教育、経営、安全管理など、関連分野への影響）

## 参照元
- 一次資料URL / 補足URL / SNS等 URL
```

### 【第3層】短縮版（SNS転用・報告用）
SNS向け、内部共有・報告用の箇条書きをまとめたファイルを1本作成する。
- **ファイル名**: `$TARGET_DIR/短縮レポート_${YEAR}-W${WEEK}.md`
- **内容**: タイトル一覧 / 箇条書き要約 / SNS投稿向け短文のまとめ

---

## ステップ 5: インフォグラフィック＆参考URL自動付与

Aランクと判定した主要な個別記事（必要に応じて週次総合まとめ等）に対して、インフォグラフィック生成スクリプトを実行する。

```bash
# Aランク記事に対する実行例
for file in "$TARGET_DIR"/A_*.md; do
    python3 .claude/generate_infographic.py "$file"
done
```
※SVG画像は指定の画像ディレクトリまたは同一ディレクトリに保存され、記事内に引用される。

---

## ステップ 6: 完了確認

1. `$TARGET_DIR` 内に何本の記事（A/B/Cランク別）およびまとめが生成されたかを報告する。
2. 「`/article-quality-check $TARGET_DIR` で全出力ファイルの品質チェック（フロントマターやタグ等）を実行してください」と促す。
