# 不足項目補完プロンプト（ニュース記事強化・Vault運用準拠）

以下の条件で、既存の短い下書きを「公開前品質チェックに耐える記事」へ拡張してください。

## 目的
- 既存ドラフトの不足項目（背景、現場示唆、論点、情報源、関連記事）を補完する。
- 事実ベースで、推測を最小化し、一次資料中心で記述する。

## 入力
1. 対象ファイルパス: `{{TARGET_PATH}}`
2. 既存本文（frontmatter含む）
3. 参照可能な一次資料URL一覧（必須）
4. 記事更新日: `{{TODAY_YYYY-MM-DD}}`

## 出力ルール
1. frontmatterは必須項目を維持する。
- `title`, `date`, `updated`, `tags`, `author`, `draft`
- `date` は原則維持、`updated` は当日に更新。
- カテゴリタグは1つだけ（既存のカテゴリタグを維持）。

2. 本文は最低1500字以上。
- 速報メモではなく、背景と実務示唆まで書く。
- 断定には根拠を添える。

3. 見出し構成（最低限）
- `## 概要`
- `## 何が起きたか（事実関係）`
- `## 背景と文脈`
- `## 飼育・運営現場への示唆`
- `## 今後の注目点`
- `## 参照元`
- `## 関連記事`

4. 参照元の要件
- 一次資料（公式発表、自治体、園館、論文、公式PDF）を優先。
- 可能なら3件以上。
- URLだけでなく「何を裏取りした情報か」を短く注記する。

5. 関連記事の要件
- `[[...]]` のウィキリンクを3件以上入れる。
- できるだけ実在ファイルに合わせる。

6. 執筆トーン
- 読者は「飼育・展示・運営に関わる実務者」も想定。
- 単なるニュース要約にせず、運用・安全・教育・広報の観点を含める。

7. 禁止事項
- 未確認情報の創作
- 日付・固有名詞の改ざん
- 根拠なしの因果断定

## 出力前セルフチェック（必須）
- [ ] frontmatter必須項目あり
- [ ] 本文1500字以上
- [ ] `## 関連記事` あり
- [ ] ウィキリンク3件以上
- [ ] 一次資料URLを3件以上明記
- [ ] 既存記事の主題から逸脱していない

## 出力形式
- 完成したMarkdown本文のみを返す（説明文は不要）。
