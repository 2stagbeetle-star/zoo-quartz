## 日次ワークフロー（毎日定時自動実行）

Windowsタスクスケジューラにより、毎日指定時刻にバックグラウンドで1日1トピックの深掘り調査が自動実行されます。

### ステップ 1 — 調査対象の登録
人間が日々のニュース等で気になったトピックを `_drafts/調査候補リスト.md` に追記（`- [ ] トピック名`）しておきます。

### ステップ 2 — リサーチと記事化の下地作成（自動）
定刻になると `run_daily_research.bat` 経由で以下のスキルが発火します。
```
/daily-topic-researcher
```
- `調査候補リスト.md` から先頭の未完了トピックを1つだけ選定。
- 一次資料の特定を含めた徹底的な深掘り調査を実施。
- `_drafts/日次リサーチ_[日付]_[トピック名].md` を出力。
- 調査対象リストを自動で `[x]` に更新。

## 週次ワークフロー（毎週月曜日）

以下のコマンドを **順番に** 実行するだけで記事生成〜公開が完了します。

### ステップ 1 — 記事生成

```
/weekly-article-generator
```

- その週の関連ニュース（動物園・水族館・自然科学全般）を網羅的に収集・調査
- ニュースの規模や質に応じて **A/B/Cランク** に分類し記事を作成
- `_drafts/YYYY-Wxx/` フォルダ内に「週次まとめ」「個別記事群（複数本）」「SNS等短縮版」の3層構造で保存（`draft: true`）

### ステップ 2 — 品質チェック

```
/article-quality-check
```

- フロントマター・タグ・文字数・ウィキリンク・関連記事を自動チェック
- レポートを `_drafts/品質チェックレポート_YYYY-MM-DD.md` に出力
- NG が 0 件になったら次のステップへ

### ステップ 3 — 記事確認・修正

Obsidian で `_drafts/YYYY-Wxx/` 内の記事群を開いて確認する:

- 事実関係の正確性と一次資料（リンク先）を確認
- 個別記事のウィキリンク `[[記事名]]` が正しいか確認
- 週次まとめやSNS用短縮版の記述をチェックし、必要に応じて加筆・修正
- `draft: true` → `draft: false` に変更（公開するファイルのみ）

### ステップ 4 — 記事アーカイブ

```
/article-archiver
```

- `draft: false` になった記事を内容・タグから自動判定し、正式フォルダ（`01_飼育日誌/` 〜 `18_学術論文・研究/`）へ移動

### ステップ 5 — デプロイ

```
/deploy
```

内部で以下が自動実行される:

1. 全公開カテゴリ（00〜18）を `~/Documents/zoo-quartz/content/` へ同期
2. ローカルビルド確認（エラーチェック）
3. GitHub へプッシュ（v4 ブランチ）
4. GitHub Actions がビルド＆Pages デプロイ（約2〜3分）

## 月次メンテナンス（毎月第1月曜日）

```

```

- 孤立ウィキリンクを検出

- 孤立リンク = 次に書くべき記事候補

```
/seasonal-topic-suggest
```

- 現在の月から次の季節トピック候補を3〜5件提案
- `_drafts/季節トピック提案_YYYY-MM.md` に保存

## トラブルシューティング

### デプロイが失敗する

```bash
# GitHub Actions のログを確認
# https://github.com/2stagbeetle-star/zoo-quartzg56/actions

# ローカルビルドエラーを確認
cd ~/Documents/zoo-quartz
npx quartz build
```

### GITHUB_TOKEN エラーが出る

```bash
# ~/.env に GITHUB_TOKEN が設定されているか確認
grep GITHUB_TOKEN ~/.env

# 未設定の場合は ~/.env に追記
echo "GITHUB_TOKEN=ghp_..." >> ~/.env
```

### ウィキリンクが壊れている

```
/link-integrity-check
```

孤立リンクの一覧を確認し、対応する記事を作成する。

### テンプレートが消えた

```
/create-templates
```

`_templates/` にテンプレートを再作成する。

### Quartz の設定をリセットしたい

```
/generate-quartz-config
```

`zoo-quartz/` ディレクトリで実行し、`quartz.config.ts` を再生成する。
