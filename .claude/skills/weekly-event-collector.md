# 週次イベント情報マルチソース収集スキル

## 概要

毎週月曜日に実行し、ウェブサイト・SNS（間接検索）・プレスリリース等の多角的ソースから生き物・自然・環境関連のイベント情報を収集する。
各イベントについて一次資料（主催者公式サイト）まで遡って裏取りを行い、`17_求人情報/生き物イベント掲示板.md` のテーブルを更新する。

---

## 実行ルール・基本方針

1. **CLAUDE.md「リサーチ運用ルール（共通方針）」に従う**
   - 1イベントずつ一次資料まで確認してから次へ進む
   - 一次資料がある場合は最優先で参照する
   - 情報源を明記できる形で残す
2. **不確実な情報は掲載しない** — 主催者・日程・場所のいずれかが確認できない場合は掲載対象外とする
3. **終了済みイベントは除外する** — 開催終了日が本日より前のものは掲載しない
4. **上限50件** — イベント数が多い場合は開催日が近い順に50件まで掲載する

## ステップ 3: 一次資料裏取り（WebFetch）

ステップ1・2で収集した各イベント候補について、以下の手順で一次資料を確認する。

1. **主催者の公式ドメインを特定する**（例: `.go.jp`, `.or.jp`, `.ac.jp`, 施設公式ドメイン）
2. **WebFetch で公式ページにアクセスし、以下を確認する:**
   - イベント名が正確か
   - 開催日程が正しいか
   - 開催場所・参加条件などの詳細
   - イベントが中止・延期されていないか
3. **一次資料に到達できた場合**: そのURLを「情報源」として採用する
4. **一次資料に到達できなかった場合**: 検索結果の二次資料URLを情報源とし、「未確認」として内部フラグを立てる（テーブルには掲載するが、一次資料確認率の集計に反映する）

**一次資料の定義:**
- 主催者の公式ウェブサイト上のページ
- 主催者の公式SNSアカウントの投稿
- 行政・自治体の公式発表
- プレスリリース配信サービス上の主催者発プレスリリース

## ステップ 4: 統合・整形・出力

### 4-1. 重複排除
同一イベント（イベント名＋主催者が同一）は1件にまとめる。URLはより信頼性の高いもの（一次資料）を優先する。

### 4-2. 終了済みイベントの除外
開催終了日が本日より前のイベントは除外する。終了日が不明の場合、開始日から14日以上経過したものは除外する。

### 4-3. 分野分類
各イベントを以下の分野に分類する:
- **爬虫類・両生類** — タイトルに「爬虫」「レプ」「両生」「トカゲ」「ヘビ」「カメ」を含む
- **アクアリウム・水生生物** — タイトルに「アクア」「水族」「イルカ」「ペンギン」「海獣」「海洋」「サンゴ」「魚」を含む
- **動物園・展示** — 動物園主催のイベント
- **自然史・博物館** — 博物館主催のイベント
- **自然保護・市民科学** — NPO主催の自然観察・保全イベント
- **公園・自然観察** — 公園主催のイベント
- **生き物・自然** — 上記に当てはまらないもの（デフォルト）

### 4-4. 区分分類
既存の区分体系を使用する:
- 博物館 / 動物園 / 市役所（自治体） / 民間 / NPO / 公園 / 学会

### 4-5. ソート順
1. 区分の順序: 博物館 → 動物園 → 市役所（自治体） → 民間 → NPO → 公園 → 学会
2. 同一区分内は開催日（開始日）昇順
3. 開催日不明のものは末尾
