# codex-reviewスキル — プラン・実装レビュー

## 概要

プラン、実装変更、またはスキルファイルを多角的な視点でレビューし、
問題点・改善点を指摘するスキルです。
指摘事項がある場合は、対象ファイルを修正してから再レビューを実施します（クリーンになるまで繰り返す）。

---

## 実行タイミング

以下のタイミングで実行すること（CLAUDE.mdのReview gateに基づく）:

- `plans/` にプランファイルを作成・更新した直後
- 主要な実装ステップ完了後（5ファイル以上変更 / 公開API変更 / インフラ設定変更）
- コミット・デプロイ前

---

## レビュー観点

### 1. 論理的一貫性
- 計画のContextとGoalsが整合しているか
- Stepsの順序が依存関係を考慮しているか
- 前後のステップに矛盾がないか

### 2. 実装可能性
- 実在するファイル・スキル・ツールを参照しているか（存在しないものを前提にしていないか）
- 各StepのWorkに具体性があり、実行可能か
- Vault全体への副作用を考慮しているか

### 3. 既存ワークフローとの整合性
- `ZOO-KNOWLEDGE-VAULT-運用ガイド.md` の日次・週次・月次ワークフローを壊さないか
- 既存12スキルとの競合・重複がないか
- フロントマター形式（title, date, updated, tags, author, draft）への準拠

### 4. 品質・安全性
- タグ体系（カテゴリタグ1記事1つ）の遵守
- `draft: false` にする前に `article-quality-check` を通しているか
- 一次資料への参照が明記されているか
- 不可逆な操作（ファイル削除・フォルダ移動）に確認ステップがあるか

---

## 処理ステップ

### ステップ 1: レビュー対象の特定

レビュー対象を以下の優先順位で決定します：
1. 直近で作成・変更されたファイルパスが指定されている場合 → そのファイルを読み込む
2. `plans/` 内の最新プランファイルをレビューする場合 → 最新の `.md` ファイルを読み込む
3. 引数なしで実行された場合 → `plans/` ディレクトリ内の全プランファイルを確認し、最新のものを選択する

### ステップ 2: 対象ファイルの読み込みと分析

- 対象ファイルを読み込みます
- 上記4つの観点でチェックを行います
- 問題・改善点を重大度別（CRITICAL / WARNING / INFO）に分類します

### ステップ 3: レビュー結果の出力

**出力先**: `_drafts/codex-review-[YYYY-MM-DD]-[対象ファイル名].md`

**出力フォーマット**:

```
# レビュー結果: [対象ファイル名]

**実行日**: YYYY-MM-DD
**対象**: [ファイルパス]

## サマリー
[問題の有無と全体評価を1〜2文で]

## 指摘事項

### CRITICAL（修正必須）
- [ ] [問題の説明] → [修正案]

### WARNING（修正推奨）
- [ ] [問題の説明] → [修正案]

### INFO（参考情報）
- [気づき・提案]

## 判定
- [ ] 再レビュー不要（CLEANと判定）
- [ ] 要修正後再レビュー
```

### ステップ 4: 修正と再レビュー（指摘事項がある場合）

- CRITICAL / WARNING の指摘がある場合、対象ファイルを修正します
- 修正後、ステップ2〜3を再実行します（re-review）
- 全ての CRITICAL が解消され、WARNINGが0件になったら「CLEAN」と判定して終了します

### ステップ 5: 完了報告

「レビュー完了: CLEAN（または修正済みCLEAN）」と出力し、レビュー結果ファイルのパスを通知します。

---

## 注意事項

- このスキルはClaude自身がレビューを行います（外部Codex連携なし）
- レビュー結果ファイル自体をレビューの対象にしないこと
- 既存記事ファイルへの直接修正は、ユーザーの確認なしには行わないこと
