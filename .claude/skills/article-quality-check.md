# 記事品質チェックスキル

## 概要
公開前の記事ファイルを品質基準で自動チェックし、レポートを出力する。
引数でファイルパスを指定するか、`_drafts/` 内の未チェックファイルを一括チェックする。

## 使い方
```
/article-quality-check [ファイルパスまたはディレクトリパス]
# 引数省略時: _drafts/ 内のすべての最新ディレクトリ・ファイルの .md をチェック
```

## チェック項目

以下の各項目を確認し、OK / NG / WARNING で評価する:

### 1. フロントマター必須項目チェック
必須フィールド: `title`, `date`, `updated`, `tags`, `author`, `draft`
- すべて存在する → OK
- 1つでも欠けている → NG（欠けているフィールド名を明示）

### 2. タグ体系準拠チェック
カテゴリタグ（以下のいずれか1つ）が含まれているか:
`飼育日誌`, `季節トピック`, `飼育ノウハウ`, `生態系ニュース`, `種別図鑑`, `自然環境`,
`水族館・水生生物`, `動物保護・保全生物学`, `獣医学・動物医療`, `生物学・生命科学`,
`生態学・自然環境学`, `植物学・植生`, `生物分類図鑑`, `環境問題・気候変動`,
`動物園学・博物館学`, `フィールドワーク・調査手法`, `求人情報`, `学術論文・研究`
- カテゴリタグあり → OK
- カテゴリタグなし → NG

### 3. ウィキリンク数チェック
`[[...]]` 形式のリンクが3つ以上あるか
- 3つ以上 → OK
- 1〜2つ → WARNING（件数を明示）
- 0 → NG

### 4. 文字数チェック
本文（フロントマターを除く）が1500字以上あるか
- 1500字以上 → OK
- 1000〜1499字 → WARNING（実際の文字数を明示）
- 1000字未満 → NG（実際の文字数を明示）

### 5. 「関連記事」セクションチェック
`## 関連記事` または `## 関連` を含むセクションがあるか
- あり → OK
- なし → NG

### 6. draft フラグチェック
`draft: true` のままになっていないか確認（公開前の最終確認用）
- `draft: false` → OK（公開準備完了）
- `draft: true` → WARNING（まだ下書き状態）

## 出力フォーマット

チェック結果を以下の形式でレポートとして表示し、`_drafts/品質チェックレポート_YYYY-MM-DD.md` に保存する:

```markdown
# 品質チェックレポート YYYY-MM-DD

## チェック対象: [ファイルパス]

| チェック項目 | 結果 | 詳細 |
|------------|------|------|
| フロントマター必須項目 | ✅ OK | 全項目あり |
| タグ体系 | ✅ OK | #飼育日誌 確認 |
| ウィキリンク数 | ⚠️ WARNING | 2個（3個以上推奨） |
| 文字数 | ✅ OK | 1823字 |
| 関連記事セクション | ✅ OK | あり |
| draftフラグ | ⚠️ WARNING | draft: true（公開前に false に変更） |

## 総合評価
- NG: 0件
- WARNING: 2件
- 公開前に対応推奨の項目: ウィキリンクを1つ追加、draft を false に変更
```

## 完了確認
- レポートのパスを表示する
- NG が 0 件の場合: 「品質チェック通過。`draft: false` に変更後、正式フォルダへ移動してください」とガイドする
- NG がある場合: 修正すべき項目を箇条書きで案内する
