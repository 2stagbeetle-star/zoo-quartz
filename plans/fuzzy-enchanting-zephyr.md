# プラン：2025年1月〜2026年4月 週次記事一括生成

## Context

現在 `_drafts/YYYY-Wxx/` フォルダ（2025-W01〜2026-W13、計64週分）が **全て空** の状態。  
`00_Index/Archive_2025/` にある `週次まとめ_` と `短縮レポート_` は25〜30行程度の簡易版であり、個別Aランク・Bランク・Cランク記事が存在しない。

`_drafts/2025-Mxx/`（月別・12ヶ月分）には5記事×12 = 60本の深掘り記事が既に完成しているが、  
週次ベースの **詳細ニュース記事群** が欠如している。

→ `/weekly-article-generator` スキルの3層構造に準拠して、64週分の記事を一括作成する。

## 各週の生成ファイル（3層構造）

### 第1層：週次総合まとめ
- **ファイル名**: `_drafts/YYYY-Wxx/週次まとめ_YYYY-Wxx.md`
- **内容**: 今週の主要動向・分類別A/B/Cランク一覧・全体傾向・現場への示唆
- Archive版より大幅に詳細化（目標400〜800字）

### 第2層：個別記事（2〜4本/週）
- **ファイル名**: `_drafts/YYYY-Wxx/[A/B/C]_[カテゴリ]_[YYYYMMDD]_[キーワード].md`
- **Aランク**: 一次資料確認済・詳細版（500字以上）
- **Bランク**: 事実確認済・通常記事（300字以上）
- **Cランク**: 速報・短報（150字以上）
- カテゴリ略称: 図鑑 / 飼育日誌 / 保全 / 生態系 / 獣医 / 生態学 / 季節 / 研究

### 第3層：短縮レポート（SNS転用版）
- **ファイル名**: `_drafts/YYYY-Wxx/短縮レポート_YYYY-Wxx.md`
- **内容**: タイトル一覧・箇条書き要約・SNS投稿用短文3案

## リサーチ方針

### 情報収集方法
- **W01〜W34（〜2025年8月）**: AIの学習データ範囲内 → 知識 + WebSearch で裏付け
- **W35〜W51（2025年9月〜12月）**: WebSearch 中心で実際のニュースを調査
- **2026-W01〜W13**: WebSearch で実際の出来事を確認

### 1トピック1リサーチの原則
- 各トピックを1件ずつ深く調査してから次へ
- 一次資料（公式発表・論文・行政資料）を優先
- 事実不明なものは掲載しない

### カバーすべき分野（毎週バランスよく）
1. 運営・社会（園館経営・法改正・JAZA動向）
2. 技術・現場（飼育技術・獣医療・繁殖管理）
3. 学術・研究（新種発見・論文・分類変更）
4. 環境・保全（気候変動・生息地・CITES）
5. 種別バラエティ（哺乳類・鳥類・爬虫類・魚類・無脊椎動物）

## 検証方法

1. 各週フォルダに3〜5ファイルが存在することを確認
2. フロントマターにdraft: true・date・tags が揃っているか確認
3. `article-quality-check` スキルで品質チェック実行
4. 既存カテゴリ記事との重複がないか確認
