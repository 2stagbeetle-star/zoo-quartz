# 週次記事の欠落調査・修正・生成プラン

## Context

Zoo Knowledge Vault の週次記事生成（`/weekly-article-generator`）が W13（2026-03-31）を最後に停止している。現在は W16（2026-04-19）であり、**W14・W15・W16 の3週分の記事が欠落**している。ユーザーの要求は「原因特定・修正・欠落分の記事作成」。

---

## 原因分析

| 原因 | 詳細 |
|------|------|
| **週次生成は手動実行が前提** | `/weekly-article-generator` は毎週月曜にユーザーが手動実行する設計。自動化されていない |
| **日次バッチも未実行** | `run_daily_research.bat` の実行ログが空。タスクスケジューラに登録されていないか、実行に失敗している |
| **調査候補リストが空** | `_drafts/調査候補リスト.md` にサンプル2件のみ。日次リサーチの入力データがない |

---

## 実行計画

### Phase 1: W14 記事生成（2026-03-30 ~ 04-05）

1. `_drafts/2026-W14/` フォルダを作成
2. WebSearch で 2026年3月30日〜4月5日の動物園・水族館・自然科学ニュースを5セグメント（運営・技術・学術・環境・種別）から網羅収集
3. 各トピックの一次資料を WebFetch で確認（1トピックずつ完了）
4. A/B/C ランク判定、既存記事との重複チェック
5. 3層構造で記事生成:
   - `週次まとめ_2026-W14.md`
   - `[A/B/C]_[カテゴリ]_[YYYYMMDD]_[キーワード].md`（複数）
   - `短縮レポート_2026-W14.md`

### Phase 2: W15 記事生成（2026-04-06 ~ 04-12）

同上の手順を W15 の日付範囲で実行。

### Phase 3: W16 記事生成（2026-04-13 ~ 04-19）

同上の手順を W16 の日付範囲で実行。本日時点までのニュースを対象とする。

### Phase 4: 自動化の修正

1. `run_daily_research.bat` に月曜日判定ロジックを追加し、月曜のみ `/weekly-article-generator` を自動実行するよう改修
2. Windows タスクスケジューラの登録状況を確認・報告
3. `_drafts/実行ログ.txt` を初期化

## 対象ファイル

| ファイル | 操作 |
|---------|------|
| `_drafts/2026-W14/` ~ `_drafts/2026-W16/` | 新規作成（記事群） |
| `00_Index/Archive_2025/` | 週次まとめ・短縮レポートのアーカイブ先 |
| `run_daily_research.bat` | 月曜判定ロジック追加 |

---

## 検証方法

1. 各週のフォルダに週次まとめ・個別記事・短縮レポートが揃っていること
2. 全記事のフロントマター（title, date, updated, tags, author, draft）が正しいこと
3. `/article-quality-check` でNG 0件になること
4. `run_daily_research.bat` の修正が構文的に正しいこと
