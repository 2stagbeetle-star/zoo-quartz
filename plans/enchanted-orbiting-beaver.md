## 安全性・リスク対策

- **git による可逆性**: Vault 全体が git 管理下にあるため、誤削除しても `git checkout` で復元可能。
- **削除順序**: 「新ファイル作成 → 古ファイル削除」の順を厳守。新ファイルが揃う前に削除しない。
- **frontmatter 規約遵守**: カテゴリタグ (`学術論文・研究`) は必ず 1 つだけ先頭付近に残す。CLAUDE.md プロジェクトルール §「カテゴリタグは1記事につき必ず1つだけ付与」を満たすため、既存のサブタグ (保全生物学, 生態学 等) はカテゴリではなく補助タグとして扱う。
- **インフォグラフィック画像**: `![[*_infographic.png]]` の参照は元ファイルと同じ名前でコピー。画像ファイル自体は触らない。
- **deploy 失敗時**: quartz build のエラーは `_drafts/deploy-log-*.txt` に記録。失敗した場合は build エラー内容を解析し、原因の記事を修正する (frontmatter YAML の整合性エラーが主因になりやすい)。
